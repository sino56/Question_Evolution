# 06. 算子组合实验方案


> 文档版本：v1.0  
> 制定日期：2026-07-11  
> 项目：Question_Evolution / PoliceQA 题目难度进化  
> 目标弱模型：Qwen3.6-27B  
> 强模型：GPT-5.4  
> 设计基线：仓库 `main` 当前主流程、`后续思路整理.md`、`样本分类画像.md` 以及本轮已确认决策。



## 1. 方案定位

算子组合不是把两个 operator 同时塞进一个超长 Prompt，而是进行可归因的顺序改写：

```text
Q0 --Oa--> Q1 --Ob--> Q2
```

第一阶段优先追求实用效果，并对少量重点 operator 对做顺序和消融实验。

## 2. 为什么最大链长为 2

- 仍可解释每一步增量效果；
- 9 个 operator 的有序双组合已经有 72 种，三组合增至 504 种；
- 每轮都会重生成题目、参考答案、rubric 和评分，链越深漂移越大；
- 便于比较 `Oa`、`Ob`、`Oa→Ob`、`Ob→Oa`。

一期禁止同一 operator 连续重复，例如 `O10→O10`。

## 3. 考虑的组合方案

### 3.1 顺序改写——推荐

优点：与现有跨轮链式机制兼容，可计算第二步增量效果。

### 3.2 联合 Prompt——暂缓

一次要求执行两个 operator，容易多主轴、难归因。

### 3.3 并行生成后融合——暂缓

成本高，融合容易变成任务堆叠。

### 3.4 条件式第二 operator——推荐增强

只有在第一步后仍存在明确、不同的剩余 failure 时，才允许第二步。

## 4. 哪些 operator 可进入组合

一期不设置固定“30% 有效率”硬阈值，采用证据排名：

```text
强制验证中的正向样本数
自然路由中的正向样本数
中位数/平均降分
硬失败数量
覆盖场景数量
Prompt 稳定性
```

选择排名靠前的 3–4 个 operator 进入组合实验。

最低证据建议但不做硬自动门禁：

- 至少在多个适配样本上完成 R>=3；
- 至少覆盖两个场景或推理子类型；
- 不主要依赖单个异常低分题；
- 没有高比例硬风险。

## 5. Operator Pair 选择

优先选择：

### 5.1 互补 failure mode

例如：

```text
O11 不可见状态归因
→ O17 行动门槛与事实门槛
```

### 5.2 同轴细化

例如：

```text
O13 新事实改变原评价
→ O15 单变量后评价保留范围
```

### 5.3 避免高度重叠

若两个 operator 只是不同名字的相同题面竞争，不进入首批组合。

## 6. 顺序效应实验

针对每个重点 pair，比较四个条件：

```text
Oa
Ob
Oa → Ob
Ob → Oa
```

使用相同：

```text
样本
根节点评分包
K/M/R
Prompt 版本
模型配置
```

## 7. 数据记录

每个组合节点保存：

```json
{
  "operator_sequence": ["O10", "O17"],
  "first_step_node_id": "...",
  "second_step_node_id": "...",
  "score_drop_from_root": 0.22,
  "incremental_drop_from_parent": 0.08,
  "prompt_length_growth": 96,
  "new_failure_mode": "...",
  "second_operator_added_unique_value": true
}
```

## 8. 一期判断方式

不使用复杂显著性阈值，记录：

```text
组合总降分
第二步增量降分
是否新增不同能力边界
是否增加硬失败或题面漂移
是否只是题目变长
```

若：

```text
Q2.score_mean < Q1.score_mean
```

则第二 operator 出现正向增量信号。

但若 Q2 与 Q1 的 target subclaim、boundary axis 完全相同，仅题长增加，则不认为形成新边界。

## 9. 组合实验 manifest

```json
{
  "experiment_id": "pair_o11_o17_v1",
  "sample_id": "6406",
  "root_node_id": "6406_root",
  "conditions": [
    ["O11"],
    ["O17"],
    ["O11", "O17"],
    ["O17", "O11"]
  ],
  "k": 3,
  "m": 2,
  "r": 3,
  "max_chain_length": 2
}
```

## 10. 第二步 operator 的条件

只有满足以下条件才继续：

- 第一步节点质量合格；
- 第一步至少 `valid_unverified`，推荐 `verified_effective`；
- 第二 failure mode 与第一步不同或确实更细；
- 不需要新增题外事实；
- chain length 尚未达到 2；
- 当前样本和分支预算允许。

## 11. 归因

必须同时记录：

```text
first_operator_effect
second_operator_incremental_effect
sequence_total_effect
```

不允许把组合全部归因给最后一个 operator，也不能全部归因给第一个 operator。

## 12. 与多边界的关系

- 若第二 operator 产生不同 boundary axis，可保存为新边界；
- 若只是同轴细化，则更新当前分支或保存更优节点；
- 新能力轴原则上更推荐从 root 独立验证，再决定是否需要组合；
- 组合不是发现所有新轴的默认方式。

## 13. 风险控制

- 多主轴：硬拒绝；
- 题长显著增加但无新 failure：标记 `complexity_only`；
- 第二步让分数升高：回第一步节点；
- 参考答案大幅重写：人工复核；
- 顺序差异过大：保留顺序特定结论，不合并成无序 pair。

## 14. 实施顺序

1. 单算子 forced/natural 数据完成；
2. 选 3–4 个 promising operator；
3. 选择 2–4 个互补 pair；
4. 每个 pair 选跨场景样本；
5. 跑四条件顺序实验；
6. 形成 `operator_pair_registry.jsonl`；
7. 只把可靠 pair 放入自然搜索的后续候选。

## 15. 验收标准

- 最大链长严格为 2；
- 完整记录 operator sequence；
- 能计算 root 总降分与第二步增量降分；
- 能比较两个顺序；
- 联合 Prompt 和融合模式未被误启用；
- 第二步无新增价值时不会因为题长增加而判有效；
- 组合失败能回到第一步节点；
- 单算子和组合报告可独立对照。
