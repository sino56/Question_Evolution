> 文档版本：v1.0  
> 制定日期：2026-07-11  
> 项目：Question_Evolution / PoliceQA 题目难度进化  
> 目标弱模型：Qwen3.6-27B  
> 强模型：GPT-5.4  
> 设计基线：仓库 `main` 当前主流程、`后续思路整理.md`、`样本分类画像.md` 以及本轮已确认决策。



## 1. 方案定位

目标是让一个原始样本可以保留多个不同能力边界题，但不引入完整树搜索。第一版采用：

```text
链式执行骨架
+ 不可变节点
+ boundary_bank
+ fork_tasks
+ 每样本最多 3 个边界
```

3 是预算上限，不是必须凑齐的产出数量。

## 2. 状态语义必须拆分

当前“有效边界”不能再直接等同于“整个样本停止”。需要两个状态：

```json
{
  "branch_status": "boundary_hit",
  "sample_search_status": "continue_other_axis"
}
```

### Branch 状态

```text
active
boundary_hit
failed
exhausted
closed
```

### Sample 状态

```text
active
continue_other_axis
completed_boundary_budget
completed_no_more_axis
stop_no_viable_boundary
manual_review
```

## 4. 边界是否重复

第一版不做复杂 embedding/LLM 去重，采用结构化近似判定：

```text
boundary_axis
operator semantic family
target_subclaim
surface_form_family
source_node
```

若以下均相同或高度相近，可视为重复：

- 同一能力轴；
- 同一最小子判断；
- 同一题面竞争结构；
- 只是实体、数字或措辞替换。

文本相似度只做辅助，不单独决定。

## 5. 分支起点规则

已确认混合规则：

### 5.1 新能力轴

```text
从 root 开新分支
```

原因：避免继承当前分支的题面偏差，更利于发现完全不同的能力边界。

### 5.2 同一能力轴细化

```text
从 best_valid_ancestor 开新分支
```

原因：保留已经验证有效的改写，只在该能力轴内继续细化。

### 5.3 无法恢复起点

优先退回 root，不创建不完整 fork。

```json
  "avoid_boundary_axes": ["unobserved_state_attribution"],
  "avoid_surface_forms": ["report_statement_competition"],
  "budget": {"k": 3, "m": 2, "r": 3, "max_depth": 2},
  "status": "pending"
}
```

## 7. Fork 触发条件

### 必须触发候选任务

- 当前节点成为新的 `boundary_hit`；
- 样本边界数小于 3；
- 仍存在未探索的适配能力轴；
- 样本未达到总调用预算；
- 不是重复边界。

### 不触发

- `score_increased`；
- 硬校验失败；
- no drop；
- assignment mismatch；
- 重复边界；
- 无剩余适配 operator；
- 样本已找到足够边界，或继续搜索价值低。

## 8. 不强制找满 3 个

样本搜索可在以下任一条件结束：

```text
找到 1–3 个有效边界，且没有新的适配能力轴
所有剩余 operator 均耗尽
root 和可靠祖先均无法产生新题
连续无新信息
达到样本预算
人工判断当前样本只适合一个边界
```

系统必须允许：

```text
0 个：暂无可发现边界
1 个：有效完成
2 个：有效完成
3 个：达到预算上限
```

## 9. 调度策略

第一版采用 FIFO + 优先级字段，不做复杂搜索：

优先级从高到低：

1. 尚未测试、且与样本 strong match 的新能力轴；
2. 已有边界但存在明确剩余 failure 的同轴细化；
3. Router 推荐但未测试的 backup；
4. possible match 探索。

每轮限制：

```text
MAX_FORK_TASKS_PER_SAMPLE = 3
MAX_FORK_TASKS_PER_ROUND = 10（建议默认）
MAX_BOUNDARIES_PER_SAMPLE = 3
MAX_CHAIN_DEPTH = 2
```

## 11. 与 Router 的关系

Fork Task 不直接绕过 Router：

- 新轴 task 可以给 Router 目标轴和 avoid 轴；
- Router 仍负责在允许集合中排序 primary/backup；
- Forced 研究模式可指定 operator；
- Natural 生产模式由 Router 选择。

## 12. Memory 语义

- 有效边界写 `boundary_bank`；
- Forced 结果先写实验级 boundary bank；
- Natural 结果经过确认后才晋升正式 bank；
- 失败仍写 failure memory，但应记录 branch/node/axis；
- 不因一个表面形态失败而禁止整个 operator。

## 14. 验收场景

### 场景一：找到一个边界后继续新轴

O11 命中，样本边界数 1，仍有 O17 strong match。

期望：O11 分支关闭，创建从 root 出发的 O17 fork task。

### 场景二：同轴细化

O17 已命中，但发现同一 action/fact 轴仍有更精确子判断。

期望：从最近 verified ancestor 开分支，而不是 root。

### 场景三：未找满 3 个即结束

只找到 1 个边界，其余轴反复无效。

期望：`completed_no_more_axis`，保留 1 个结果。

### 场景四：重复边界

新候选仅换实体和措辞。

期望：不新增 boundary，不创建下一 fork。

## 15. 验收标准

- `boundary_hit` 不再自动终止整个样本；
- 一个样本可保存多个 boundary record；
- 边界数最多 3；
- 新轴从 root、同轴细化从 best valid ancestor；
- 重复边界不会重复计数；
- Fork Task 可断点恢复、不会重复执行；
- 样本可以在 0、1、2 或 3 个边界时合理结束；
- 生产与实验 boundary bank 隔离。
