# Question_Evolution 详细方案包


> 文档版本：v1.0  
> 制定日期：2026-07-11  
> 项目：Question_Evolution / PoliceQA 题目难度进化  
> 目标弱模型：Qwen3.6-27B  
> 强模型：GPT-5.4  
> 设计基线：仓库 `main` 当前主流程、`后续思路整理.md`、`样本分类画像.md` 以及本轮已确认决策。



本方案包将后续建设拆成八份可独立评审、又可组合实施的正式方案。设计目标不是一次性重写现有流水线，而是在保持现有 `run_loop.sh` 主体顺序和历史数据兼容性的前提下，逐步把项目从“单链、单候选、单次评分”升级为：

```text
可版本化 Prompt 体系
+ 每算子 K 道候选
+ 预选 M 道进入真实压测
+ 每题 R 次重新作答与评分
+ 评分后最终选择
+ 完整祖先节点与多层回退
+ 一个样本多个能力边界
+ 推理形态与业务场景适配
+ 最大链长 2 的受控算子组合
```

## 已确认的关键决策

1. 一个原始样本可以保留多个不同能力边界题，建议最多 3 个；3 是上限，不是必须完成的数量。
2. 题目进化专门针对 Qwen3.6-27B；GPT-5.4 作为强模型和保护性可回答性检查模型。
3. 初期效果判断保持简单：候选题多次评分平均分低于父题多次评分平均分，即记录正向降分信号；暂不设置最低降分幅度。
4. Qwen3.6-27B 同时承担弱模型作答者和 judge，这是刻意设计，一期保持不变。
5. 重复评分固定参考答案和 rubric，每次重新生成 Qwen 答案，再由 Qwen judge 评分。
6. Seed 支持情况不确定，系统不得把不同 seed 视为独立性保证。
7. 首轮参数：`K=3`、`M=2`、`R=3`，边界或高波动候选可扩展到 `R=5`；并发数 10。
8. `R=1` 只作为临时筛选；`R>=3` 且平均分下降后才记录正式正向信号。
9. 第一轮 O10–O18 Prompt 验证同时保留两种模式：
   - 适配性约束下的强制验证；
   - Router 自然路由端到端验证。
10. 强制验证与自然路由使用独立样本池、独立 Memory、独立报告，不互相污染。
11. 第一轮只优化实际主流程使用的 active operator Prompt stack，固定 profile、router、validator 和评分逻辑。
12. Operator 的能力语义保持稳定；Prompt 写法可以版本化迭代，语义变化必须升级 semantic version 或新增 operator。
13. Prompt 最终采用“公共基础 Prompt + operator spec + reasoning adapter + scenario adapter + modifiers”的分层结构。
14. 多层回退采用：单次硬失败回父节点；累计失败回最近 `verified_effective` 祖先；无可靠祖先时回 root。
15. 新能力轴默认从 root 开新分支；同一能力轴细化从 `best_valid_ancestor` 开新分支。
16. 算子组合优先使用受控顺序改写，最大链长 2，并比较顺序效应。
17. 样本画像当前是 LLM 初步分类；先复核 50 条，其中随机 30 条、低置信或冲突 20 条。
18. 场景采用 `primary_scene + secondary_scenes[]`；流程/规范型继续拆成 7 个子类型。
19. 禁止引入外部知识，所有 adapter 只能使用样本现有题面、答案、标签和元数据。
