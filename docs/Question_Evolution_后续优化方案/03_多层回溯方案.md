# 03. 多层回溯方案


> 文档版本：v1.0  
> 制定日期：2026-07-11  
> 项目：Question_Evolution / PoliceQA 题目难度进化  
> 目标弱模型：Qwen3.6-27B  
> 强模型：GPT-5.4  
> 设计基线：仓库 `main` 当前主流程、`后续思路整理.md`、`样本分类画像.md` 以及本轮已确认决策。



## 1. 方案目标

将当前“直接父题单层快照”升级为可恢复完整祖先关系的节点系统，并在失败时执行：

```text
直接父节点
→ 最近 verified_effective 祖先
→ root
```

一期不实现完整树搜索调度，但数据结构必须支持未来分支。

## 2. 三种可选实现

### 2.1 方案 A：仅祖先链

- 每个样本只有一个 active node；
- 保存完整 parent 指针；
- 失败沿链回退；
- 改造最小。

### 2.2 方案 B：祖先链 + 有界分支队列

- 方案 A 的节点模型；
- 额外保存 boundary bank 和 fork task；
- 命中边界后可从 root/祖先开新分支；
- 最符合“一个样本多个能力边界”。

### 2.3 方案 C：完整搜索图

- active frontier、节点优先级、BFS/DFS/UCB；
- 工程复杂、数据需求大；
- 当前暂缓。

### 2.4 推荐

```text
数据模型按方案 B 设计
第一版回退执行按方案 A 实现
单算子和节点机制稳定后开启 B 的 fork task
```

## 3. 不可变节点模型

每个题目版本形成一个节点：

```json
{
  "node_id": "6406_b01_n02",
  "sample_id": "6406",
  "root_node_id": "6406_root",
  "parent_node_id": "6406_b01_n01",
  "branch_id": "6406_b01",
  "depth": 2,
  "prompt": "...",
  "operator_sequence": ["O11", "O17"],
  "evaluation_bundle_id": "eval_...",
  "node_status": "verified_effective"
}
```

历史节点不被覆盖。回滚只更新样本级：

```text
active_node_id
best_valid_ancestor_id
```

## 4. 节点状态

| 状态 | 含义 |
| --- | --- |
| `root_safe` | 原始题或已准入根节点 |
| `generated_unvalidated` | 已生成，未完成校验 |
| `valid_unverified` | 质量合格，但未完成 R>=3 |
| `verified_effective` | R>=3 且平均分低于父题 |
| `soft_failed` | 合格但 no drop / full score no drop 等软失败 |
| `hard_failed` | 不可回答、硬校验失败、score increased 等 |
| `boundary_hit` | 已作为有效能力边界保存 |
| `superseded` | 不再作为 active，但保留审计 |

一期 `verified_effective` 定义保持简单：

```text
quality passed
AND R >= 3
AND node.score_mean < parent.score_mean
```

## 5. 失败分类

### 5.1 硬失败

- 不可回答；
- 复杂度/事实/泄漏硬校验失败；
- `score_increased`；
- Prompt 漂移至原题外能力轴；
- GPT-5.4 明确无法回答。

### 5.2 软失败

- `no_clear_effect`；
- `full_score_no_drop`；
- 合格但平均分不下降；
- 模板重复但未触发硬风险。

### 5.3 不立即计失败

- `needs_manual_review`；
- R 不足；
- assignment mismatch；
- API 暂时异常。

## 6. best_valid_ancestor 定义

`best_valid_ancestor` 不是全历史最低分题，而是：

> 从当前节点沿 parent 链向上，距离最近、质量合格、R>=3 且平均分低于其父节点的可靠祖先。

选择顺序：

```text
最近 verified_effective
→ root_safe
```

一期不采用复杂效用函数，避免历史最低分节点因偶然、漂移或过度狭窄而被误选。

## 7. 默认回退规则

```text
单个硬失败
→ 回直接父节点，切换 operator

同一父节点累计 2 个硬失败
或 1 个硬失败 + 2 个软失败
→ 回最近 verified_effective 祖先

连续漂移、无可靠祖先、祖先预算耗尽
→ 回 root

root 仍无可用 operator
→ stop_no_viable_boundary
```

`needs_manual_review` 不立即增加硬/软失败计数。

## 8. 回退算法

```python
def choose_rollback_target(current_node, sibling_attempts, node_store):
    parent = node_store.get(current_node.parent_node_id)

    if current_node.failure_class == "hard" and not parent.exhausted:
        return parent.node_id, "retry_from_parent"

    stats = sibling_attempts.for_parent(parent.node_id)
    if stats.hard_failures >= 2 or (
        stats.hard_failures >= 1 and stats.soft_failures >= 2
    ):
        ancestor = find_nearest_verified_effective(parent, node_store)
        if ancestor:
            return ancestor.node_id, "rollback_to_best_valid_ancestor"

    root = node_store.get(current_node.root_node_id)
    if not root.exhausted:
        return root.node_id, "rollback_to_root"

    return None, "stop_no_viable_boundary"
```

## 9. Operator 切换

回退后必须：

- 避免刚刚失败的 operator；
- 避免同一 `operator + surface_form_family + failure_type` 的重复；
- 优先使用 `recommended_next_methods`；
- 不惩罚整个 operator 的所有表面形态；
- 在同一父节点记录 sibling attempt，防止重复试验。

## 10. 数据持久化

### 10.1 `evolution_nodes.jsonl`

保存所有不可变节点。

### 10.2 `node_attempts.jsonl`

保存：

```text
parent_node_id
operator_id
prompt_version
surface_form_family
failure_class
attempt_status
```

### 10.3 样本级状态

```json
{
  "sample_id": "6406",
  "root_node_id": "6406_root",
  "active_node_id": "6406_b01_n02",
  "best_valid_ancestor_id": "6406_b01_n01",
  "sample_search_status": "active"
}
```

## 11. 与当前 parent_snapshot 的迁移

迁移策略：

1. 当前 prompt 创建 active node；
2. 若存在 `parent_snapshot`，创建直接父节点；
3. 无更早历史时，将父节点标为 `root_safe` 或 `legacy_parent_unknown_depth`；
4. 保留原 `parent_snapshot` 作为 fallback，暂不删除；
5. 新节点流程稳定后，再将 `parent_snapshot` 降级为兼容字段。

## 12. run_loop 集成

建议在最终选择后：

```text
candidate_final_selection
→ create_evolution_node
→ collect_answers / rubric / scoring 或复用已完成候选闭环
→ analyze_effect
→ update_node_status
→ backtrack_controller
→ update sample_search_state
```

如果候选已经在 K×M×R 阶段完成新参考答案、rubric 和评分，则最终选择后不应再次重复调用。

## 13. 验收场景

### 场景一：直接父节点回退

```text
Q0 → Q1 有效 → Q2 硬失败
```

期望：active 回 Q1，避开 Q2 使用的 operator。

### 场景二：连续 sibling 失败

```text
Q1 → Q2-A 硬失败
Q1 → Q2-B 硬失败
```

期望：回 Q1 的最近 verified ancestor；若 Q1 自身即 verified，则留在 Q1 切换新 operator family。

### 场景三：软失败累积

```text
1 硬 + 2 软
```

期望：触发祖先回退。

### 场景四：无可靠祖先

期望：回 root；root 耗尽后停止。

## 14. 测试要求

- 节点不可覆盖；
- parent/root 指针完整；
- 不产生循环；
- 最近 verified ancestor 选择正确；
- assignment mismatch 不增加失败计数；
- manual review 不立即回退；
- score increased 恢复父节点对应评分包；
- 回退后 Router 避开失败 operator/surface form；
- 历史 JSONL 无节点字段时仍可运行。

## 15. 验收标准

- 能从任意 active node 恢复完整祖先链；
- 回滚目标由节点状态决定，不再只依赖 `prompt_old`；
- 同一父节点失败次数可审计；
- 多层回退不会丢失 references、rubric、score_prompt 和 evaluation bundle；
- root 停止条件明确；
- 全过程支持断点续跑。
