# 08. 分阶段实施路线图

## 1. 总体原则

后续开发不得一次性实施：

```text
Prompt 重构
+ K×M×R
+ 多层回溯
+ 多边界分支
+ Adapter
+ 算子组合
+ 统一复杂 Schema
```

应按依赖关系逐步实施，每个阶段都必须：

- 有 baseline；
- 只验证少量假设；
- 有 feature flag；
- 有回滚条件；
- 有退出标准；
- 不通过时停止继续叠加后续机制。

## 2. 阶段 0：冻结与基线

### 目标

保留当前严格版本，并建立可复现 baseline。

### 工作内容

- 标记当前主流程为 `legacy_strict`；
- 固定模型、Prompt 和 scoring 版本；
- 保存一批代表性样本；
- 建立 baseline 实验报告；
- 记录当前真实评分覆盖率、降分率和调用成本。

### 退出条件

- 同一输入可重复运行；
- baseline 结果可追溯；
- 关键配置有 hash；
- 旧版本可一键运行。

## 3. 阶段 1：恢复最小核心闭环

### 目标

建立 `core` 模式。

### 工作内容

- 跳过非致命前置门禁；
- 增加统一 fatal check；
- 保留答案、rubric、scoring；
- 候选级失败隔离；
- 所有结果落盘。

### 退出条件

- core 可独立运行；
- 单候选失败不终止批次；
- 所有非致命候选可进入首次评分；
- 可计算 score drop。

## 4. 阶段 2：Shadow Validator

### 目标

保留现有 validator 分析价值，但取消其默认否决权。

### 工作内容

- 将 validator 输出写入 observations；
- 增加 `would_reject_in_strict_mode`；
- 比较 core、shadow、strict；
- 统计 false reject 与 false accept。

### 退出条件

- shadow 与 core 的 R=1 候选数一致；
- strict false reject 可统计；
- 没有 validator 能在无数据支持下自动晋升硬门禁。

## 5. 阶段 3：K×R 候选实验

### 目标

提高 operator 实验稳定性。

### 工作内容

- 每 operator 生成 K=3；
- 所有非致命候选 R=1；
- exploitation + exploration 补齐 R=3；
- 高波动候选可扩展 R=5；
- 评分后最终选择。

### 退出条件

- K 个候选均可追踪；
- R 状态明确；
- 父题缓存可复用；
- 能计算 operator 批次效果；
- 不存在评分前静态 K→M 强淘汰。

## 6. 阶段 4：稳定数据层

### 目标

为后续树搜索建立不可变数据基础。

### 工作内容

- Node、Edge、Candidate；
- Trial、Bundle；
- 配置 hash；
- append-only event；
- 原子写入；
- 断点续跑；
- namespace 隔离。

### 退出条件

- 重跑不会重复 node/trial；
- 配置错配可发现；
- 历史节点不被覆盖；
- 所有核心不变量测试通过。

## 7. 阶段 5：轻量树搜索

### 目标

支持多节点和局部回溯。

### 工作内容

- Beam Width=2；
- Max Depth=2；
- frontier；
- 搜索预算；
- 局部回溯；
- 一个样本多个 boundary；
- tree feature flag。

### 暂不实施

- BFS/DFS/UCB；
- 强化学习策略；
- 多级 failure counter；
- 自动 operator blacklist；
- 复杂 fork scheduler。

### 退出条件

- 搜索不会循环；
- 回溯不覆盖历史；
- 可在预算达到后停止；
- tree 关闭后 core 正常；
- 一个边界命中后可继续其他轴。

## 8. 阶段 6：离线算子组合

### 目标

验证第二个 operator 是否有真实增量。

### 工作内容

- 选择少量 operator pair；
- 跑 Oa、Ob、Oa→Ob、Ob→Oa；
- 最大链长 2；
- 计算第二步增量；
- 独立 namespace；
- 不写回 Natural Router。

### 退出条件

- 四条件可比较；
- 第二步增量可计算；
- 组合失败不影响单算子；
- 没有增量的 pair 不进入主搜索。

## 9. 阶段 7：有限组合接入树搜索

### 目标

仅将被证实有效的 pair 作为条件式搜索动作。

### 工作内容

- `ENABLE_OPERATOR_CHAIN`；
- pair registry 仅作建议；
- 第二步触发条件；
- 搜索预算；
- 组合与单算子效果分开统计。

### 退出条件

- chain 关闭后 tree 正常；
- 最大链长严格为 2；
- 组合不会显著提高 fatal rate；
- 有稳定正向第二步增量。

## 10. 阶段 8：后验 Curator

### 目标

构建正式数据集质量控制。

### 工作内容

- 对 confirmed boundary 做严格可回答性检查；
- 事实一致性；
- rubric 稳定性；
- 答案泄漏；
- 格式负担；
- 人工复核；
- 导出训练集、评测集和研究案例集。

### 退出条件

- Explorer 与 Curator 状态分离；
- 被拒绝进入数据集的候选仍保留实验结果；
- 数据准入不会反向改变搜索历史。

## 11. Adapter 的位置

Adapter 不应作为前期必选阶段。

只有在以下问题被数据确认后再考虑：

- 同一 operator 在不同场景中持续模板化；
- Router 能力轴选择基本稳定；
- 现有 Prompt 难以利用场景上下文；
- 仅修改 operator Prompt 无法解决。

第一版 Adapter 只能作为软 Prompt 上下文，不拥有硬准入权。

## 12. 每阶段统一验收模板

每阶段必须提交：

```text
目标假设
baseline
修改范围
新增复杂度
实验样本
结果指标
真实评分覆盖率
score drop 变化
调用成本变化
false reject 变化
失败案例
回滚方法
是否继续下一阶段
```

## 13. 停止条件

如果某阶段出现以下情况，应停止继续叠加复杂机制：

- 真实评分覆盖率下降明显；
- score drop 没有改善；
- 每个边界成本显著上升；
- 新增状态无法稳定复现；
- 多个模块职责重叠；
- 需要继续增加规则才能让新机制工作；
- core baseline 被破坏；
- feature flag 无法完整关闭新机制。

## 14. 推荐最终里程碑

### Milestone A：可复现最小闭环

```text
core + shadow + strict
```

### Milestone B：稳定多候选实验

```text
K×R + evaluation bundle + false reject analysis
```

### Milestone C：轻量多节点搜索

```text
immutable node + beam search + local backtrack + multi-boundary
```

### Milestone D：受控算子组合

```text
offline pair experiment + conditional chain length 2
```

### Milestone E：正式数据整理

```text
posthoc curator + dataset export
```

只有完成前一个 Milestone 的稳定验收，才进入下一个。
