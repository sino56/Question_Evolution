# 07. 数据结构、稳定运行与验收方案

## 1. 设计目标

数据结构和工程稳定性应服务于实验可追溯性，而不是固化尚未验证的复杂规则。

核心要求：

- 稳定 ID；
- 不可变历史；
- 配置一致；
- 幂等执行；
- 原子写入；
- 候选级失败隔离；
- 可断点续跑；
- forced、natural 和 production 隔离；
- 可统计真实 score drop 与 validator false reject。

## 3. 暂缓固化的字段

以下字段在实验尚未证明必要前，不应进入核心必填 schema：

- hard_failure_count；
- soft_failure_count；
- best_valid_ancestor_id；
- remaining_candidate_axes；
- operator_exhausted；
- adapter_incompatible；
- compatibility whitelist；
- global operator penalty；
- complex priority score；
- fork task 多级状态；
- complete parent stack；
- auto-elimination status。

可先作为可选观察字段。

## 4. 稳定 ID

ID 不得依赖数组位置。

推荐：

```text
node_id =
sample_id + prompt_hash_prefix

candidate_group_id =
parent_node_id + operator_id + prompt_version + assignment_id

candidate_id =
candidate_group_id + generation_replica_id

evaluation_bundle_id =
node_id + config_hash

evaluation_trial_id =
evaluation_bundle_id + trial_index
```

## 5. 配置 Hash

配置 hash 至少包括：

```text
prompt hash
reference hash
rubric hash
score prompt hash
answer model
judge model
temperature
top_p
trial rule
scoring code version
operator Prompt version
```

同一个 node 在配置不同的情况下必须创建不同 bundle。

## 6. 不可变与 Append-only

不可原地覆盖：

- node prompt；
- parent relation；
- reference；
- rubric；
- score prompt；
- trial answer；
- trial score；
- operator sequence。

修正方式：

```text
创建新 node
创建新 bundle
写新的 event
```

状态索引可以更新，但必须保留事件日志。

## 7. 幂等执行

相同输入和相同 config hash：

```text
优先命中已有 artifact
不得重复生成 node
不得重复写 trial
不得重复写 boundary
```

每个阶段必须支持：

```text
run_if_missing_or_incomplete
```

但“文件存在”不是唯一判断，应同时检查：

- schema；
- 记录数；
- ID 覆盖；
- config hash；
- completion status。

## 8. 原子写入

正式文件写入流程：

```text
写入 output.tmp
→ schema 校验
→ 数量和 ID 覆盖检查
→ fsync
→ 原子替换正式 output
```

Memory、boundary 和 search state 不应在处理中直接写正式主文件。

## 9. 错误隔离

### 候选级

- 单次 API 失败；
- 单个输出解析失败；
- 单候选参考答案失败；
- 单候选 rubric 失败。

处理：

```text
有限重试
→ candidate_failed
→ 继续其他候选
```

### 阶段级

- 输入损坏；
- 父节点丢失；
- ID 冲突；
- 大量 candidate 与 parent 错配；
- trial 配置错位；
- 正式输出无法保证一致性。

处理：

```text
停止本阶段
不得发布部分正式结果
```

## 10. 实验隔离

推荐 namespace：

```text
experiments/core/*
experiments/shadow/*
experiments/forced/*
experiments/natural/*
experiments/tree/*
experiments/operator_pair/*
experiments/production_curator/*
```

Memory：

```text
memory/forced/*
memory/natural/*
memory/tree/*
memory/operator_pair/*
memory/production/*
```

禁止自动跨 namespace 合并。

## 13. 方案验收指标

### 稳定运行

- 完整实验批次可运行；
- 单候选失败不影响其他候选；
- 断点续跑不重复产物；
- 配置错配可被发现；
- tree/chain 开关可独立关闭。

### 研究有效性

- 非致命候选首次评分覆盖率接近 100%；
- 能计算真实 score drop；
- 能发现 validator false reject；
- 树搜索不会减少 core baseline 的可复现性；
- 算子组合结果不污染单算子评价。

### 工程复杂度

每阶段报告：

```text
新增文件数
新增配置数
新增状态数
新增硬门禁数
新增模型调用数
删除或降级规则数
```

## 14. 完成标准

当以下条件满足时，系统可认为达到稳定基础版本：

1. core、shadow、strict 可切换；
2. K×R 流程稳定；
3. evaluation bundle 可缓存和复用；
4. 不可变节点可追溯；
5. 候选级错误隔离；
6. validator false reject 可统计；
7. tree search 可关闭；
8. operator chain 可关闭；
9. 所有核心不变量测试通过；
10. 旧数据可通过 fallback 读取，但不会迫使新架构继续累积兼容状态。
