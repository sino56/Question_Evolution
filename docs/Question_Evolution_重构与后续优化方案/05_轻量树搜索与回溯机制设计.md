# 05. 轻量树搜索与回溯机制设计

## 1. 方案定位

树搜索的目标是：

- 从多个已评分节点中选择下一次探索起点；
- 在不同 operator、能力轴和表面形态之间分配预算；
- 避免单链流程在某一方向失败后无法继续；
- 支持一个样本发现多个不同能力边界。

树搜索不是新的生成流水线，而是位于 Core Executor 上方的调度器。

## 2. 设计原则

1. 历史节点不可修改；
2. 回溯只改变下一次展开节点；
3. 第一步采用有界搜索，不实现完整 BFS、DFS、UCB 或强化学习；
4. 搜索判断依赖真实评分；
5. 单次失败只局部影响当前节点和当前生成配置；
6. 树搜索必须可以通过 feature flag 完全关闭。


```json

  "operator_prompt_version": "o17_v3",
  "generation_replica_id": 2,

```

## 5. 节点状态

第一版只保留：

```text
root
generated
scored
effective
no_drop
fatal_invalid
archived
```

不建议第一版引入：

```text
valid_unverified
soft_failed
hard_failed
rollback_pending
ancestor_exhausted
operator_exhausted
superseded
stop_no_viable_boundary
```

状态越多，越容易产生无法验证的组合逻辑。

## 6. 第一版搜索策略

推荐使用有界 Beam Search：

```text
MAX_DEPTH = 2
BEAM_WIDTH = 2
OPERATORS_PER_NODE = 1
K = 3
R_INITIAL = 1
R_CONFIRMED = 3
MAX_NODE_EXPANSIONS_PER_SAMPLE = 12
MAX_BOUNDARIES_PER_SAMPLE = 3
```

流程：

```text
选择 frontier 节点
→ 选择一个 operator
→ 生成 K 个候选
→ 所有非致命候选 R=1
→ exploitation + exploration 补齐 R=3
→ 形成新节点
→ 更新 frontier
```

## 7. Frontier 选择

第一版不必实现复杂数值公式，可以采用配额法：

```text
70% 预算：当前 score drop 最大的有效节点
20% 预算：不同能力轴或不同表面形态节点
10% 预算：随机或高不确定性节点
```

目的：

- 防止搜索完全贪心；
- 保留对新能力轴的探索；
- 估计 Router 和 validator 的偏差；
- 减少陷入单一模板。

## 8. 回溯语义

回溯不是恢复旧文件，也不是覆盖当前节点。

例如：

```text
Q0 → Q1 → Q2
```

如果 Q2 无降分：

```text
Q2 保留为 no_drop
下一次 expansion source 切回 Q1
```

所有历史结果继续存在。

## 9. 第一版回溯规则

### 9.1 子节点有效

```text
child.score_mean < parent.score_mean
→ child 进入 frontier
```

### 9.2 子节点无降分

```text
child.score_mean >= parent.score_mean
→ child 标记 no_drop
→ parent 仍可尝试其他 operator 或其他 replica
```

### 9.3 当前父节点全部候选无降分

```text
返回最近 effective 父节点
→ 尝试其他 operator 或 surface form
```

### 9.4 父节点预算耗尽

```text
回 root
→ 尝试新能力轴
```

### 9.5 root 预算耗尽

```text
结束当前样本搜索
```

第一版不使用：

- 两个硬失败触发回退；
- 一个硬失败加两个软失败；
- 连续漂移阈值；
- operator 耗尽状态机；
- 多层 penalty score。

## 10. 局部失败作用域

失败记录必须限定为：

```text
parent_node_id
operator_id
operator_prompt_version
surface_form_family
failure_type
```

允许：

```text
同一个 parent 下，暂时避免完全相同的 operator + Prompt version + surface form
```

禁止：

```text
某个样本一次失败
→ 全样本永久禁止 operator
→ 全局降低 operator 权重
```

## 11. 多边界语义

找到一个边界后：

```text
branch_status = boundary_hit
sample_search_status = continue_other_axis
```

一个样本可保存 0–3 个边界。

新能力轴优先从 root 开始；同轴细化可以从最近 effective 节点继续。

第一版不必实现复杂 fork scheduler。可以先由 Search Controller 直接生成下一条扩展任务。

## 12. 搜索事件

所有调度决策写入 append-only 日志：

```json
{
  "event_id": "...",
  "sample_id": "...",
  "event_type": "select_expansion_source",
  "source_node_id": "...",
  "selected_operator_id": "O17",
  "reason": "best_score_drop_with_remaining_budget",
  "budget_snapshot": {},
  "created_at": "..."
}
```

## 13. Feature Flags

```text
ENABLE_TREE_SEARCH=false
ENABLE_BACKTRACK=false
ENABLE_MULTI_BOUNDARY=false
MAX_TREE_DEPTH=2
BEAM_WIDTH=2
MAX_NODE_EXPANSIONS_PER_SAMPLE=12
```

关闭后必须退回普通链式模式。

## 14. 稳定性测试

必须覆盖：

1. 节点不可覆盖；
2. parent/root 指针有效；
3. 不产生循环；
4. 回溯只改变 active source；
5. no_drop 节点仍保留；
6. 单样本失败不写全局 blacklist；
7. 搜索预算能够停止；
8. tree search 关闭后 core 正常；
9. 重启后不会重复展开同一个已完成任务；
10. 一个边界命中后仍可探索其他能力轴。

## 15. 验收标准

- 能从 root 生成两层有界搜索树；
- 所有节点有真实评分或明确 fatal 状态；
- 回溯不会删除历史；
- 每个决策可追溯；
- 搜索不会无限循环；
- 可在预算达到时稳定停止；
- 一个样本可保存多个边界；
- 关闭树搜索后不影响原始闭环。
