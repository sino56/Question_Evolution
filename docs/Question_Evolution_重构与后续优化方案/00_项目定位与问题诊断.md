# 00. 项目定位与问题诊断

## 1. 项目核心目标

`Question_Evolution` 的目标不是生成形式上更长、更复杂、更规范的题目，而是：

1. 使用 Qwen 对原始题目作答并评分；
2. 使用 GPT 分析原题、参考答案、rubric、Qwen 回答和得分；
3. 选择适合的进化算子；
4. 生成新的、更可能暴露 Qwen 能力不足的题目；
5. 为新题生成参考答案、rubric 和 score prompt；
6. 再次使用 Qwen 作答和评分；
7. 比较进化前后的得分变化；
8. 根据具体失分项归因分析 Qwen 的能力边界。

核心效果指标为：

```text
score_drop = parent_score_mean - candidate_score_mean
```

- `score_drop > 0`：候选产生正向降分信号；
- `score_drop = 0`：未观察到能力变化；
- `score_drop < 0`：进化后题目反而更容易。

一期不必设置复杂的最低降分阈值。只要在相同模型和评分配置下，候选平均分低于父题平均分，就应记录为正向信号。

## 2. 当前问题的本质

当前项目后期出现的问题，不是某一个 validator 写得过严，而是系统目标逐渐从“发现能力边界”转向“避免生成失败题”。

典型演化过程为：

```text
实验出现失败
→ Codex 分析失败原因
→ 增加一条规则避免再次失败
→ 增加阈值、状态、Memory 或回退
→ 更多候选在真实评分前被拦截
→ 系统继续为拦截结果增加解释和兼容逻辑
```

最终主流程容易变成：

```text
样本画像
→ 路由
→ 多候选生成
→ 复杂度检查
→ 可回答性检查
→ 事实检查
→ 难度收益预测
→ 候选选择
→ 参考答案与 rubric
→ Qwen 真实评分
```

这形成了一个根本逻辑倒置：

> 先由 GPT 和静态规则预测一道题是否可能让 Qwen 失败，通过后才允许真正测试 Qwen。

## 3. 目标函数漂移

项目原始目标是：

```text
最大化真实能力边界发现率
```

后期可能被替换为：

```text
最大化 validator 通过率
最小化候选生成失败
最小化格式风险
最小化状态异常
保证每轮流程全部成功
```

这些工程指标只能作为辅助指标，不能替代真实 score drop。

项目不应将以下情况默认视为软件缺陷：

- 某个算子没有产生降分；
- 进化后得分升高；
- Router 选错了 operator；
- GPT 对难度的预测不准确；
- 同一个算子在不同样本上表现不一致；
- 多个候选中只有一个有效；
- 一道题需要人工复核。

这些都属于实验结果。

## 4. 探索系统与数据质检系统混淆

能力边界探索与最终数据生产具有不同目标。

### 4.1 Boundary Explorer

目标是高召回地发现能够让 Qwen 暴露不足的题目。

特点：

- 允许失败；
- 允许不确定候选进入真实评分；
- 允许一定比例的低质量候选；
- 依赖真实 score drop 判断效果；
- 所有失败都应保留用于分析。

### 4.2 Dataset Curator

目标是从已证明有效的边界题中筛选正式数据。

需要检查：

- 是否可回答；
- 是否存在关键事实缺失；
- 是否有答案泄漏；
- rubric 是否稳定；
- 降分是否来自合理能力难度；
- 是否只是格式负担、题外知识或不可回答导致降分。

因此：

> 严格校验可以决定题目能否进入最终数据集，但不能决定题目是否有资格参加能力边界实验。

## 5. 正确的治理方向

后续优化应从“继续加规则”转向“重新划分系统职责”。

需要建立以下边界：

1. 生成模块负责扩大探索空间；
2. Qwen 真实评分负责判断题目是否有效；
3. validator 负责提供风险标签和分析线索；
4. 数据整理器负责最终准入；
5. 搜索控制器负责选择下一次从哪个节点继续；
6. 历史节点和评分结果不可覆盖。

## 6. 判断一项优化是否合理

一项优化只有在满足以下条件时才值得进入主流程：

1. 解决的是多轮实验中重复出现的高频问题，而不是单个案例；
2. 有清晰的 baseline；
3. 能通过同样本、同配置的 A/B 实验证明净收益；
4. 不会显著降低非致命候选的真实评分覆盖率；
5. 不会将 GPT 预测替代为最终效果证据；
6. 可以通过 feature flag 关闭；
7. 有明确回滚条件；
8. 新增复杂度低于它节省的实验成本。

## 7. 后续核心优化方向

按优先级排序：

1. 恢复稳定最小闭环；
2. 增加 core、shadow、strict 三种运行模式；
3. 所有非致命候选先完成一次真实评分；
4. 使用 R 次重复作答与评分降低偶然性；
5. 建立不可变节点和稳定 evaluation bundle；
6. 逐步引入轻量树搜索；
7. 回溯仅改变展开起点；
8. 算子叠加先做独立对照实验；
9. 最后再决定是否需要 Adapter、复杂 Memory 或自动算子淘汰。
