# 02. 目标架构与模块职责设计

## 1. 设计目标

目标架构需要同时满足：

1. 最小实验闭环可独立运行；
2. Qwen 真实评分是主要效果证据；
3. validator 默认不阻止首次评分；
4. 树搜索和回溯是可插拔调度能力；
5. 算子叠加不破坏单算子归因；
6. 历史节点、Prompt 和评分不可覆盖；
7. 系统支持断点续跑、缓存与失败隔离；
8. 最终数据质量控制与边界探索分离。

## 2. 总体架构

```text
                    ┌────────────────────────────┐
                    │      Search Controller     │
                    │ node/operator/budget/queue │
                    └─────────────┬──────────────┘
                                  │
                                  v

│ Parent Node │ → │ Candidate Gen │ → │ Fatal Check Only│
└─────────────┘   └───────────────┘   └────────┬────────┘
                                               │
                                               v

                                  │      Core Executor     │
                                  │ answer/rubric/Qwen eval│
                                  └───────────┬────────────┘
                                              │
                         ┌────────────────────┴────────────────────┐
                         │                                         │
                         v                                         v
             ┌─────────────────────┐                   ┌────────────────────┐
             │  Posthoc Observer   │                   │   Search Update    │
             │ risk/quality/labels │                   │ frontier/backtrack │

             └─────────────────────┘                   └────────────────────┘
```

## 3. Core Executor

### 3.1 职责

Core Executor 负责完成一条候选从生成结果到真实评分的稳定闭环：

```text
候选题
→ 参考答案
→ rubric
→ score prompt
→ Qwen 作答
→ Qwen judge 评分
→ evaluation bundle
→ score drop
```

### 3.2 不负责

Core Executor 不负责：

- 决定使用哪个 operator；
- 决定从哪个祖先节点展开；
- 决定是否创建新分支；
- 判断候选是否进入正式数据集；
- 更新全局 operator 权重；
- 进行复杂树搜索。

### 3.3 核心接口

```text
evaluate_candidate(
    parent_node,
    candidate_prompt,
    operator_metadata,
    evaluation_config
) -> evaluation_bundle
```

核心输出至少包括：

```text
candidate_id
parent_node_id
prompt
reference_answer
rubric
score_prompt
trial_results
score_mean
score_std
score_min
score_max
score_drop
config_hash
```

## 4. Search Controller

### 4.1 职责

Search Controller 负责：

- 选择下一次从哪个节点展开；
- 选择 operator；
- 分配候选生成预算；
- 决定哪些候选从 R=1 扩展到 R=3 或 R=5；
- 管理 frontier；
- 执行局部回溯；
- 控制最大深度、宽度和调用预算；
- 决定是否继续其他能力轴。

### 4.2 权限边界

Search Controller 不得：

- 修改历史节点；
- 修改历史评分；
- 覆盖旧 Prompt；
- 把一次失败转成全局 operator blacklist；
- 删除实验失败候选；
- 直接决定最终数据集准入。

## 5. Observer

### 5.1 职责

Observer 为候选和评分结果增加旁路分析：

- 多主轴；
- 输出任务数；
- 题长；
- 格式复杂度；
- 外部知识风险；
- 事实来源风险；
- 答案路径泄漏；
- 可回答性；
- 与父题能力轴偏移；
- 与历史候选重复；
- GPT 预测难度收益；
- 降分来源分析。

### 5.2 默认行为

在 `shadow` 模式中：

```text
Observer 只写 observations
不得改变候选进入首次真实评分的数量
```

输出示例：

```json
{
  "observations": {
    "multi_axis": true,
    "format_complexity_risk": "medium",
    "external_fact_risk": "low",
    "predicted_difficulty_gain": 0.58,
    "would_reject_in_strict_mode": true
  }
}
```

## 6. Dataset Curator

### 6.1 职责

Curator 只处理已完成真实评分的候选，用于判断：

- 是否进入正式评测数据；
- 是否进入训练数据；
- 是否仅作为边界研究案例；
- 是否需要人工复核；
- 是否因不可回答或错误事实而不适合发布。

### 6.2 输出状态

```text
accepted_for_dataset
accepted_for_boundary_analysis_only
manual_review
rejected_from_dataset
```

即使候选被拒绝进入最终数据集，也不能删除它的实验记录。

## 7. Artifact Store

### 7.1 核心要求

Artifact Store 必须提供：

- 不可变节点；
- 稳定 ID；
- append-only trial；
- evaluation bundle；
- 配置 hash；
- Prompt 版本；
- 父子边；
- 搜索事件；
- 缓存；
- 原子写入；
- 断点续跑。

### 7.2 推荐存储对象

```text
samples.jsonl
nodes.jsonl
edges.jsonl
candidates.jsonl
evaluation_trials.jsonl
evaluation_bundles.jsonl
observations.jsonl
boundaries.jsonl
search_events.jsonl
curation_results.jsonl
```

第一阶段不必立即拆成所有文件，但语义应保持独立。

## 8. 运行模式

### 8.1 core

```text
最小闭环
+ 致命错误检查
+ 所有非致命候选首次真实评分
```

### 8.2 shadow

```text
core
+ 全部 Observer
+ Observer 不改变评分覆盖率
```

推荐作为能力边界实验默认模式。

### 8.3 tree

```text
shadow
+ Search Controller
+ 不可变节点
+ frontier 和局部回溯
```

### 8.4 tree_with_chain

```text
tree
+ 经过验证的最大链长 2 算子组合
```

### 8.5 strict_curator

```text
对真实评分后的候选执行严格后验质检
```

## 9. 模块依赖原则

允许的依赖：

```text
Search Controller → Core Executor
Core Executor → Artifact Store
Observer → Artifact Store
Curator → Artifact Store
Search Controller → Observer summary
```

不允许的反向依赖：

```text
Core Executor 不依赖树搜索状态
Core Executor 不依赖 Curator 结果
Observer 不得控制 Core Executor 首次评分
Curator 不得修改 Search Controller 历史结果
```

## 10. 架构不变量

1. core 模式永远可独立运行；
2. shadow 开关不得改变非致命候选首次评分数量；
3. 回溯不得覆盖节点；
4. forced 与 natural 实验隔离；
5. 父题与候选配置不一致时不得计算正式 score drop；
6. 单样本失败不得污染全局 operator；
7. tree search 关闭后，不影响普通链式实验；
8. operator chain 关闭后，单算子结果仍可复现。
