# 06. 算子叠加与组合实验方案

## 1. 方案定位

算子叠加的目标不是简单增加题目复杂度，而是研究：

- 一个 operator 已产生某种困难后；
- 第二个 operator 是否能进一步暴露新的或更细的能力不足；
- 不同顺序是否产生不同效果。

组合实验必须建立在稳定单算子实验之后。

## 2. 前置条件

在开展组合实验前，应满足：

1. 单算子 core/shadow 流程稳定；
2. 至少有一批 operator 在多个样本上完成真实评分；
3. Prompt version 固定；
4. 父题和候选评分配置一致；
5. evaluation bundle 可复用；
6. 节点和边数据结构已经稳定；
7. 组合实验与 Natural Router Memory 隔离。

不要求使用固定“30% 有效率”硬阈值，但需要有足够真实数据。

## 3. 顺序改写

禁止：

```text
在同一个 Prompt 中同时执行 Oa 和 Ob
```

推荐：

```text
Q0 --Oa--> Q1 --Ob--> Q2
```

每一步独立生成：

- 题目；
- 参考答案；
- rubric；
- score prompt；
- Qwen 回答；
- Qwen 评分。

这样才能计算第二步增量。

## 4. 最大链长

一期固定：

```text
MAX_OPERATOR_CHAIN_LENGTH = 2
```

禁止：

```text
Oa → Ob → Oc
```

原因：

- 归因困难；
- 题面漂移增加；
- 参考答案和 rubric 多次重写；
- 调用成本快速膨胀；
- 搜索空间过大。

## 5. 四条件对照

针对一个 operator pair，比较：

```text
Oa
Ob
Oa → Ob
Ob → Oa
```

保持一致：

- 样本；
- root；
- Prompt version；
- K；
- R；
- answer model；
- judge；
- 温度；
- top_p；
- scoring 版本。

## 6. 效果计算

### 6.1 第一步效果

```text
first_operator_drop = Q0.score_mean - Q1.score_mean
```

### 6.2 第二步增量效果

```text
second_operator_incremental_drop =
Q1.score_mean - Q2.score_mean
```

### 6.3 根节点总效果

```text
sequence_total_drop =
Q0.score_mean - Q2.score_mean
```

只有：

```text
second_operator_incremental_drop > 0
```

才能说明第二个 operator 具有正向增量。

如果 Q2 仅保持 Q1 的低分，而没有进一步下降，不能判定第二个 operator 有效。

## 7. 第二步 operator 的最低条件

第二步只需满足：

1. Q1 已完成真实评分；
2. chain length 小于 2；
3. 第二步能力目标与第一步不同，或是明确的同轴细化；
4. 当前预算允许；
5. 候选能通过致命错误检查。

不要求：

- Adapter 必须 strong match；
- Router 和 GPT 同时同意；
- operator pair 已进入硬白名单；
- 所有 validator warning 均为低；
- operator 历史成功率达到固定阈值。

这些可以作为观察信息。

## 9. 新边界与同轴细化

### 新边界

第二步形成不同能力轴：

```text
保存新的 boundary record
```

### 同轴细化

第二步仍在相同轴，但压测更精确子判断：

```text
保留 Q1 和 Q2
根据 score drop、稳定性和后验质量决定主节点
```

不能因为 Q2 更低就删除 Q1。

## 10. 组合实验与主流程隔离

第一阶段使用独立 namespace：

```text
memory/operator_pair_pilot/*
experiments/operator_pair/*
```

不得：

- 直接更新 Natural Router 权重；
- 写入 production operator blacklist；
- 自动把成功 pair 加入主搜索动作；
- 因组合失败降低单算子评价。

## 11. 组合接入树搜索的条件

只有当某个 pair：

- 在多个样本上有正向第二步增量；
- 不主要依赖题长增加；
- 不产生高比例致命错误；
- 在至少两个场景或推理类型中有证据；
- 顺序结论稳定；

才可以作为条件式搜索动作。

接入后仍必须通过：

```text
ENABLE_OPERATOR_CHAIN
```

控制。

## 12. 风险控制

以下风险默认作为观察：

- 多主轴；
- 题长显著增加；
- 事实变化过多；
- 参考答案大幅重写；
- rubric 大幅变化；
- 不同顺序效果差异大。

只有致命错误才阻止首次评分。

## 13. 验收标准

1. 最大链长严格为 2；
2. 能比较四种条件；
3. 能计算第一步、第二步增量和总效果；
4. 第二步无增量时不会误判成功；
5. 组合失败不影响单算子评价；
6. 组合实验与 Natural Memory 隔离；
7. 关闭组合开关后树搜索仍正常；
8. 所有 operator sequence 可完整追溯。
