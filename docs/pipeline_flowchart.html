评分阶段：受控并行架构

样本 worker、Qwen 请求池和 GPT 请求池分别配置，默认上限均为 20。Qwen 回答和 judge 共享同一请求池，避免在途请求叠加。

有界任务调度与流式 I/O

Reader 仅在队列有容量时提交记录，Writer 批量落盘。内存、协程和待写结果限制在并发配置附近，不随输入线性增长。

原子发布与中断恢复

正式产物只在阶段完整成功后才可见。中断后可从 checkpoint 恢复，避免重复计算。

Memory 索引与本地校验复用

倒排索引 + 匹配缓存避免全量扫描 memory bank；规则版本一致时复用进化阶段的校验结果。

⚙

样本级并行处理

有界 worker 池让多个样本同时处理，替代原有逐条串行。worker 数量固定，内存和协程不随输入增长。

逐条串行 N worker 并行
⚖

模型请求公平调度

Qwen/GPT 分别维护公平请求池，跨样本优先分配名额。避免少数慢题独占额度，降低尾部等待。

样本并发 × 采样数 = 突发 全局请求上限 + 公平分配
⚡

样本内 Trial 并行

同一样本的独立回答 trial 和评分 repeat 在依赖满足后并行执行，结果按索引归位，不改变语义。

trial 串行排队 trial 并行 + 归位
☰

流式读写 + 批量落盘

逐行读取 JSONL 跳过空行、即时报错。Writer 按记录数/字节/时间阈值批量 flush，减少 I/O 等待。

全量加载 + 逐条 flush 流式读取 + 批量写入
♺

中断恢复 & Checkpoint

partial + checkpoint 记录已确认进度。中断重启时跳过已完成记录/候选组，从断点继续。

中断 = 整轮重跑 断点续跑
✔

计算复用 & Trace 外置

memory 倒排索引 + 缓存避免全量扫描；同规则版本校验结果直接复用；原始响应移至 sidecar 减轻主记录负担。

全量扫描 + 重复计算 索引 + 缓存 + 复用
